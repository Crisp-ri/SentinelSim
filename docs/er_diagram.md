# Database Schema

The system uses a relational SQLite database with three main tables.

---

## events

Stores all incoming security events.

Fields:

- id (Primary Key)

- timestamp (event occurrence time)

- source_ip (origin of the event)

- target (target system or service)

- port (target port number)

- event_type (type of event)

- payload (raw event data)

Indexes:

- index on timestamp

- index on source_ip + timestamp

- index on event_type

---

## alerts

Stores alerts generated by the Detection Engine.

- id (Primary Key)

- alert_type (type of threat detected)

- severity (threat level)

- created_at (alert creation time)

---

## ip_stats

Stores aggregated statistics per IP address.

Fields:

- ip (PK)

- total_events

- suspicious_score

- first_seen

- last_seen

- updated_at

---

# Relationships

---

- One event can generate multiple alerts

- Each alert is linked to a specific event

- IP statistics aggregate multiple events
